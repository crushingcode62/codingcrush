NAMED EXCEPTIONS

The exceptions which are already given names by PL/SQL are declared in the STANDARD package in PL/SQL. You do not have to declare them in your own programs. SQLCODE is a PL/SQL built-in function that returns the status code of the last-executed statement. SQLCODE returns zero if the last statement executed without errors.
PL/SQL provides many pre-defined exceptions, which are executed when any database rule is violated by a program. For example, the predefined exception NO_DATA_FOUND is raised when a SELECT INTO statement returns no rows.

Syntax:

DECLARE
< declarations section>
BEGIN
< executable command(s)>
EXCEPTION
< exception handling goes here >
WHEN exception1 THEN
exception1-handling-statements
WHEN exception2 THEN
exception2-handling-statements
WHEN exception3 THEN
exception3-handling-statements
........
WHEN others THEN
exception-handling-statements
END;


Some of the named exceptions are :

Exception Name Oracle Error Error Description
NO_DATA_FOUND ORA-01403 It is raised when a select into statement returns no rows.
VALUE_ERROR ORA-06502 It is raised when an arithmetic, conversion, truncation, or size-constraint error occurs.
ZERO_DIVIDE ORA-01476 It is raised when an attempt is made to divide a number by zero.
TOO_MANY_ROWS ORA-01422 It is raised when a SELECT INTO statement returns more than one row.
ROWTYPE_MISMATCH ORA-06504 It is raised when a cursor fetches value in a variable having incompatible data type.
DUP_VAL_ON_INDEX ORA-00001 It is raised when duplicate values are attempted to be stored in a column with unique index.
