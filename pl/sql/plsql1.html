PL/SQL BLOCK STRUCTURE

PL/SQL is a block-structured language. This means that programs can be divided into logical blocks. A PL/SQL block consists of up to three sections:

  1. Declaretion
  2. Executeble
  3. Exception handling

1. Declaration :The declaration section of a PL/SQL block start with the keyword DECLARE. This section is used to declare any placeholders like variables, constants, records and cursors, which are used to manipulate data in the executable section.

2. Executable: The executable section of a PL/SQL block starts with the keyword BEGIN and ends with END. This is the section where the program logic is written to perform any task. The programmatic constructs like loops, conditional statement and SQL statements from the part of execution section.

3. Exception Handling: The Exception section of a PL/QL block starts with the keyword EXCEPTION. Any error in the program can be handled in this block, so that the PL/SQL block terminates gracefully. If the PL/SQL block contains exceptions that cannot be handled, the block terminates immediately with errors.


The structure of PL/SQL block:

DECLARE
< declarations>
BEGIN
< executable statement>
EXCEPTION
< exception handlers>
END;
