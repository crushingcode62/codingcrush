EXPLICIT CURSOR

Explicit cursors are declared explicitly by the user, along with other identifiers to be used in a pl/sql block. These are also known as user defined cursors, defined in the declare section of the pl/sql block. They must be declared when you are executing a select statement that returns more than one row. Even though the cursor stores multiple recordes, only one record can be processed at a time, which is called as current row.

EXPLICIT CURSOR ATTRIBUTES

Oracle provides few attributes for explicit cursor called as explicit cursor attributes to check the status of DML operation.
These attributes are:

  1. %FOUND
  2. %NOTFOUND
  3. ROWCOUNT
  4. %ISOPEN

1. %FOUND

True, if fetch statement returns at least one row.
False, if fetch statement doesn't return a row. Example:
cursor_name%FOUND

2. %NOTFOUND

False, if fetch statement returns at least one row.
True, if fetch statement doesn't return a row. Example:
cursor_name%NOTFOUND

3. %ROWCOUNT

Return the number of rows fetched by the fetch statement. If no row is returned, the pl/sql statement returns an error. Example:
cursor_name%ROWCOUNT

4. %ISOPEN

True, if the cursor is already open in the program. False, if the cursor is not opened in the program. Example:
cursor_name%ISOPEN

Cursor manipulation

Cursor manipulation is done through the declare, open, fetch and close statements. This statements are explained below:

Declaring an Explicit cursor
While declaring a cursor, cursor name is given and also defines the structure of the query to be performed within it. The Cursor statement is used to declare explicit cursor. Syntax :

CURSOR < cursor_name> IS < select statement>;


Opening an Explicit cursor
Once the cursor is declared in the declaration section, we can access the cursor in the execution section of the pl/sql program. The open statement is used to open an explicit cursor.
After opening the cursor, the rows returned by the query are available for fetching. When a cursor is opened, the first row becomes the current row i.e. the cursor points to the first row in the active set. Syntax:

OPEN cursor_name;


Fetching data from the Explicit cursor
After the cursor is opened, the current row is loaded into variables. The current row is the row at which the cursor is currently pointing. The transfer of data into pl/sql variablesis done through fetch statement. Syntax:

FETCH cursor_name into variable;


Closing an Explicit cursor
Close the cursor after completing the processing of the select statement. This step allows the cursor to be reopened, if required. Therefore, you can establish an active set several times. The close statement is used to close an explicit cursor. Syntax:

CLOSE cursor_name;
