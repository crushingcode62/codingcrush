TRIGGER LEVEL

1. ROW LEVEL TRIGGER

A row-level trigger is fired each time the table is affected by the triggering statement, i.e. it is fired once for each row that is affected by the triggering statement.
Row-level triggers are useful if the code in the trigger action depends on data provided by the triggering statement or rows that are affected. Row level triggers are implemented by using the option FOR EACH ROW in CREATE TRIGGER statement.

Syntax for a ROW-LEVEL TRIGGER :

CREATE [OR REPLACE] TRIGGER trigger_name
BEFORE | AFTER
INSERT [OR]| UPDATE [OR]| DELETE [of column_name]
ON table_name
FOR EACH ROW [when condition]
BEGIN
pl/sql block;
END trigger_name;

2. STATEMENT LEVEL TRIGGER

A statement-level trigger is fired once on behalf of the triggering statement, regardless of the number of rows in the table that the triggering statement affects.

Syntax for a STATEMENT-LEVEL TRIGGER :

CREATE [OR REPLACE] TRIGGER trigger_name
BEFORE | AFTER
INSERT [OR]| UPDATE [OR]| DELETE [of column_name]
ON table_name
BEGIN
pl/sql block;
END trigger_name;

TRIGGER TIMING

1.BEFORE TRIGGER

When defining a trigger, you can specify the trigger timming. That is, you can spacify whether the trigger action is to be executed before or after the triggering statement. Before and after apply to both statement and row level.
BEFORE triggers are commonly used to check the validity of the data before the action is performed. For instance, you can use BEFORE trigger to prevent deletion of row if deletion should not be allowed in the given case.

Syntax for a BEFORE TRIGGER :

CREATE [OR REPLACE] TRIGGER trigger_name
BEFORE
INSERT [OR]| UPDATE [OR]| DELETE [of column_name]
ON table_name
FOR EACH ROW [when condition]
BEGIN
pl/sql block;
END trigger_name;

2. AFTER TRIGGER

AFTER triggers are fired after the triggering action is completed. For example, if after trigger is associated with insert command then it is fired after the row is inserted into the table.

Syntax for AFTER TRIGGER :

CREATE [OR REPLACE] TRIGGER trigger_name
AFTER
INSERT [OR]| UPDATE [OR]| DELETE [of column_name]
ON table_name
FOR EACH ROW [when condition]
BEGIN
pl/sql block;
END trigger_name;
