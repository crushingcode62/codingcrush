CONSTRAINTS

Constraints:SQL uses integrity constraints to prevent invalid data entry into the tables of the database. Constraints are a part of the table definition that is used to limit the values entered into its columns. Integrity constraints provide a way of ensurung that changes made to the database by authorized users do not result in a loss of data consistency.
While creating a table, constraints can be placed on the values that will be entered into its column(s). SQL will reject any value that violates the criteria that were defined. Constraints can be defined in two ways:

  1. Column level constraints:The constraints can be specified immediately after the column definition.This is called column-level constraints.
  2. Table level constraints: The constraints can be specified after all the columns are defined.This is called table-level constraints.The table level constraints are defined at the end of the create table statement which can impose rules on any column in the table.

Note: Constraints can be specified when a table is created (with the create table command) and after the table is created(with the alter table command).

Types of constraints:

  1. Primary Key
  2. Foreign Key
  3. Not NULL
  4. Unique
  5. Check
  6. Default
